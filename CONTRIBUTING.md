# Contributing

Thanks for your interest in the American Cloud Python SDK.

## This SDK is generated

All code under `src/americancloud/` is auto-generated by
[Fern](https://buildwithfern.com) from the American Cloud OpenAPI
specification. Generation happens in our internal platform repository — not
here — so **pull requests that edit the generated source can't be merged**;
any change would be overwritten by the next regeneration.

What that means in practice:

- **Found a bug or missing field in the SDK?** It almost always traces back to
  the API or its OpenAPI document. [Open an issue](../../issues) describing the
  endpoint and the behavior you expected — we fix it at the source and the fix
  flows into the next release of every SDK language.
- **Hand-maintained files** (this file, `README.md`, `pyproject.toml`,
  `LICENSE`, `CHANGELOG.md`, `VERSIONING.md`, the release workflow) are the
  exception — improvements to docs or packaging are welcome as PRs.

## Building locally

```bash
python -m pip install build
python -m build          # produces sdist + wheel in dist/
pip install dist/*.whl   # install into your environment
```

## Releases

Releases are cut by tagging `vX.Y.Z`; GitHub Actions builds and publishes to
PyPI via trusted publishing. The SDK version matches the American Cloud API
platform version it was generated from — see [`VERSIONING.md`](./VERSIONING.md).

## Questions
